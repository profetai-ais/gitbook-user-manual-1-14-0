# 個人化

## 簡介

個人化是一項讓 Agent 能在後續互動中「記住」並「延續使用」關鍵資訊的功能，用來提升對話的連貫性與任務處理效率。當使用者在互動過程中提供偏好、背景資料、常用設定、專案脈絡或固定格式等資訊時，系統可將這些內容整理為可重用的記憶，讓 Agent 在之後的對話中自動參照，減少重複說明與來回確認。

例如，使用者常用的稱呼、語氣風格、輸出格式（條列/表格/段落）、常處理的產品或專案名稱、流程偏好、常見決策原則，以及需要長期維持一致的規範等，都可以作為記憶被保留並在適當時機套用。

記憶功能的入口在右上角頭像的「個人化」分頁。

### 記憶

記憶分為「儲存記憶」與「參考記憶」。

<figure><img src="../.gitbook/assets/image (128).png" alt=""><figcaption></figcaption></figure>

* **儲存記憶**：開啟時，允許 Agent 儲存記憶。
* **參考記憶**：開啟時，允許 Agent 在回應時引用記憶庫裡的記憶。

## 管理記憶

管理記憶的入口在「管理」按鈕，記憶管理分為「個人記憶」與「Agent 記憶」。

<figure><img src="../.gitbook/assets/image (129).png" alt=""><figcaption></figcaption></figure>

### 個人記憶

當參考記憶的功能開啟時，儲存在此的記憶能被所有 Agent 調用。個人記憶條目最多可儲存 50 條記憶。

> Note：當超過儲存上限時，系統將自動刪除使用頻率最低的記憶條目。

<figure><img src="../.gitbook/assets/image (130).png" alt=""><figcaption></figcaption></figure>

### Agent 記憶

當參考記憶的功能開啟時，Agent 能調用屬於自己的記憶庫內的記憶。 每位 Agent 最多可儲存 50 條記憶條目。

> Note：當超過儲存上限時，系統將自動刪除使用頻率最低的記憶條目。

## 記憶的儲存方法

<figure><img src="../.gitbook/assets/image (133).png" alt=""><figcaption></figcaption></figure>

<figure><img src="../.gitbook/assets/image (134).png" alt=""><figcaption></figcaption></figure>

<figure><img src="../.gitbook/assets/image (135).png" alt=""><figcaption></figcaption></figure>

1. 請確認使用記憶功能前已開啟「儲存記憶」。
2. 在對話的介面選擇對話的 Agent，並輸入想記憶的內容。
3. 記憶成功後， Agent 會提示可以到「個人化 → 記憶管理」查看記憶。
