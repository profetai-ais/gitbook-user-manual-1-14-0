# 排程

## 簡介

<figure><img src="../.gitbook/assets/image (385).png" alt=""><figcaption></figcaption></figure>

Schedule 可讓使用者建立排程任務，讓指定的 Agent 依照設定的時間或週期自動執行指令。使用者可以透過 Schedule 定期執行例行任務，例如每日摘要、定時通知、資料查詢、報表產出或自動化流程。

透過排程設定，使用者不需要手動開啟 Agent 對話，也能讓 Agent 在指定時間自動執行任務，並保留每次執行的結果與紀錄。

## 建立排程

### 從排程管理頁面建立

<figure><img src="../.gitbook/assets/image (381).png" alt=""><figcaption></figcaption></figure>

<figure><img src="../.gitbook/assets/image (382).png" alt=""><figcaption></figcaption></figure>

<figure><img src="../.gitbook/assets/image (384).png" alt=""><figcaption></figcaption></figure>

1. 切換到排程分頁
2. 點選右上角 新增 可開啟 新增排程 視窗，建立新的排程任務
3. 輸入排程名稱
4. 選擇觸發排程的 Agent，一旦選擇後則不能變更
5. 選擇排程執行模式
   1. 獨立模式 : 每次排程執行皆視為獨立任務，適合不需要累積上下文的單次執行情境
   2. 上下文模式 : 排程會延續並累積執行脈絡，適合需要持續追蹤、累積資訊或延續任務狀態的情境
6. 輸入 Agent 每次執行時要完成的任務內容，指令應盡量明確描述任務目標、輸出格式與需要注意的條件，以提高排程執行結果的穩定性
7. 填入時間區間，系統會依照所選時區計算排程執行時間
8. 選擇排程的時間週期，排程支援以下三種週期模式 :
   1. 週期執行 : 用於設定固定間隔重複執行的排程。例如設定為**每30分鐘**，表示排程會從**起始時間**開始，每 30 分鐘自動執行一次
   2. 指定時間 : 用於設定排程指定執行的日期與時間。使用者需選擇執行日期與時間。此模式適合只需在特定時間執行一次，或依系統設定於指定時間觸發的任務
   3. 自訂 : 用於設定較複雜的排程規則。使用者可以輸入自然語言描述，讓系統協助產生 Cron，也可以直接填入 Cron 表達式
9. 選擇執行上限。至少需啟用一項使用限制。當任一已啟用的使用限制達到設定的門檻值時，排程將自動停用。排程支援以下停止條件 :
   1. Date : 設定結束日期。到達指定日期後，排程將停止執行
   2. Count : 設定最大執行次數。達到指定次數後，排程將停止執行
   3. Token : 設定最大 Token 使用量。達到指定 Token 數量後，排程將停止執行
10. 設定通知對象，使用者可以設定排程完成後是否通知指定使用者，目前排程支援信箱通知以及 webhook 模式 :
    1. Email 通知 : 系統會透過電子郵件通知指定收件人
    2. Webhook : 使用者可以設定外部 URL。排程執行完成後，系統可將結果以 POST 方式傳送至指定的外部服務
11. 設定完成後點擊新增，完成設定

### 從 Agent 設定頁面建立

<figure><img src="../.gitbook/assets/image (386).png" alt=""><figcaption></figcaption></figure>

入口位於 Agent 設定頁面的排程管理，新增的流程可參考 [從排程管理頁面建立](pai-cheng.md#cong-pai-cheng-guan-li-ye-mian-jian-li) ，唯一不同的是，在 Agent 設定頁面不可再另外設定其他 Agent 。



## 查看日誌

在排程詳細頁面中，點選**日誌**可查看該排程的執行紀錄。

<figure><img src="../.gitbook/assets/image (380).png" alt=""><figcaption></figcaption></figure>

日誌清單會顯示每次執行的標題與狀態。使用者可以點選紀錄查看該次執行的詳細內容。

執行結果會以對話形式呈現，包含 Agent 的回覆內容、執行成本與相關操作。使用者可以複製執行結果，或返回日誌清單查看其他紀錄。

日誌可協助使用者確認排程是否成功執行、Agent 回覆內容是否符合預期，以及每次執行的結果是否需要後續處理。



## 注意事項

排程會依設定的時區與週期自動執行。建立前請確認時間區間、起始時間、週期與執行上限條件是否正確。

若排程執行失敗，可至日誌查看錯誤紀錄與執行結果。若排程未如預期執行，請確認排程是否啟用、停止條件是否已達門檻，以及 Agent 是否仍可正常使用。

若排程任務會呼叫外部服務或寄送通知，請確認 Webhook URL、通知方式與收件人設定是否正確。
