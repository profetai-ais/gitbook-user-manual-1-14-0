# AI Studio 權限架構介紹

## 權限架構

<figure><img src="../.gitbook/assets/image (397).png" alt=""><figcaption></figcaption></figure>

### 功能層級（Root）&#x20;

用來管理「某一類功能」的使用與管理權限，例如 Agent 清單、知識庫清單、模板清單....等。這一層決定你能不能進入該功能、能不能建立新項目、能不能管理該功能的成員名單等。

Root 的檢視權限取決於 AI Studio 的角色權限配置，AI Studio 的角色權限說明請參照 [AI Studio 角色權限介紹](ai-studio-jue-se-yu-quan-xian-shuo-ming.md)。

### 項目層級（Object）&#x20;

用來管理功能內「每一個單獨項目」的權限，例如某一個 Agent、某一個知識庫、某一個模板。這一層讓你可以針對單一項目設定成員角色，決定誰能編輯、誰只能使用、誰可以管理成員等。

Object 的檢視權限取決於 Root 的角色權限配置，Root 的角色權限說明請參照 模組權限角色介紹 的各個[清單角色權限](quan-xian-gong-neng-jie-shao.md)

> 注意：預設情況下，只有具備管理權限的角色可以進入對應功能與進行管理操作。若你無法看到某些功能或無法執行操作，請向管理員確認你在「功能層級」與「項目層級」的權限是否已被授權。

## 權限判斷方式

系統會先判斷使用者在該功能中的 Root 權限，再依照使用者的 Root 角色決定是否需要進一步判斷 Object 權限。

* 若使用者是該功能的**清單管理員**，則可管理該功能中的物件，即使沒有個別物件的 Object 權限，也能執行新增、編輯與刪除等操作。
* 若使用者是該功能的**清單協作者**或**清單使用者**，則仍需依照 Object 權限判斷可存取與可操作的物件範圍。
* 若使用者沒有該功能的 Root 權限，通常無法進入該功能頁面，也無法操作該功能中的物件。

### 注意事項

使用者實際可見與可操作的內容會依 Root 權限與 Object 權限設定而有所不同。若使用者無法看到特定功能或物件，請確認該使用者是否具備對應功能的 Root 權限，以及該物件是否已授予對應的 Object 權限。

若使用者需要管理整個功能中的所有物件，可將該使用者設定為該功能的 Root Admin。若只需授權使用者操作特定物件，則可透過 Object 權限進行設定。

調整權限後，部分頁面可能需要重新整理，或重新進入功能頁面後才會顯示最新的權限結果。



## Root 權限

<figure><img src="../.gitbook/assets/image (347).png" alt=""><figcaption></figcaption></figure>

Root 權限是整個模組最外層的控制權限，是用於控制使用者在某個功能中的整體操作範圍。Root 權限通常分為以下角色：

* **清單管理員**
* **清單協作者**
* **清單使用者**

不同 Root 角色可執行的操作範圍會依功能設計而有所不同。

若使用者具備某項功能的 Root 權限，才可以進入該功能頁面，或使用該功能提供的主要操作。例如進入 Agent、知識庫、模板 或其他功能模組。

其中，**清單管理員**具有該功能下的最高管理權限。當使用者是某個功能的清單管理員時，即使未被授予特定 Object 權限，也可以對該功能中的物件進行新增、編輯與刪除等管理操作。

## **Object 權限**

<figure><img src="../.gitbook/assets/image (348).png" alt=""><figcaption></figcaption></figure>

Object 權限用於控制使用者在功能內可存取與操作的個別物件。

同一個功能中可能包含多個物件，例如不同的 Agent、知識庫、模板或其他項目。當使用者不是該功能的清單管理員時，系統會依照每個物件的 Object 權限，判斷使用者是否可以查看、使用、編輯或管理該物件。

## AI Studio 與 Root 權限同步

登入後，系統會自動將清單權限補上 AI Studio 帳號當下的權限。

例如 : 帳號為 **AI Studio 協作者**，登入後自動補上 Agent、知識庫、能力(技能、MCP)、 模板(工作流程、提示詞)、分析(報告) 的**清單協作者**權限。

此功能**只會在登入時觸發同步當前沒有權限的部分**，**不會影響現有的清單權限**，如果該帳號在 AI Studio 的權限有異動，不會觸發自動同步權限功能。
