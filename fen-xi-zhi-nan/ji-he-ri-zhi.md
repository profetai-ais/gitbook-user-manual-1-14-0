# 稽核日誌

## 簡介

稽核日誌用於記錄系統內重要物件的操作紀錄，協助管理者追蹤使用者在平台中的新增、編輯與刪除等行為。透過稽核日誌，使用者可以查看操作時間、操作人員、物件類型、物件名稱與異動內容，方便進行系統管理、問題追蹤與資料稽核。

## 欄位說明

| 欄位   | 說明                                         |
| ---- | ------------------------------------------ |
| 操作類型 | 顯示被操作的物件類型，例如 Agent、數據集、技能、工作流程模版、提示詞模板 等。 |
| 操作名稱 | 顯示被操作的物件名稱。點選名稱可查看該筆操作的詳細異動內容。             |
| 動作   | 顯示操作類型，例如 CREATE、UPDATE、DELETE。            |
| 操作人  | 顯示執行該操作的使用者。                               |
| 建立日期 | 顯示該筆操作紀錄建立的時間。                             |

## 查看稽核日誌內容

<figure><img src="../.gitbook/assets/image (377).png" alt=""><figcaption></figcaption></figure>

使用者可以點選清單中的操作名稱開啟稽核日誌預覽視窗，查看該筆操作的詳細內容。

在**變更前**與**變更後**區塊中，系統會顯示該筆操作造成的資料差異。使用者可以透過此內容確認物件在操作前後的變更狀況。

不同操作類型會顯示不同的異動內容。例如，**新增** 操作通常會顯示新增後的資料內容；**更新** 操作會顯示修改前後的差異；**刪除** 操作則可用於追蹤被刪除物件的相關資訊。

## 搜尋與篩選

<figure><img src="../.gitbook/assets/image (378).png" alt=""><figcaption></figcaption></figure>

使用者可以透過頁面上方的搜尋欄輸入名稱，快速查找特定物件的操作紀錄。

也可以使用篩選功能縮小查詢範圍，例如依物件類型、操作類型、操作人員或時間區間查詢紀錄。篩選條件可協助使用者快速找到特定期間或特定使用者的操作紀錄。

## 匯出稽核日誌

若需要保存或進一步分析稽核紀錄，使用者可以點選右上角的**匯出**按鈕開啟匯出稽核日誌視窗。

<figure><img src="../.gitbook/assets/image (392).png" alt=""><figcaption></figcaption></figure>

匯出前可設定以下條件：

| 欄位           | 說明         |
| ------------ | ---------- |
| 操作類型         | 選擇要匯出的物件類型 |
| 動作           | 選擇要匯出的操作類型 |
| 操作名稱         | 輸入指定物件名稱   |
| 操作人          | 選擇執行操作的使用者 |
| 開始時間 \~ 結束時間 | 設定要匯出的時間範圍 |

完成條件設定後，可選擇以下格式匯出：

* **匯出 CSV**：匯出為 CSV 檔案，適合以試算表工具檢視或整理。
* **匯出 JSON**：匯出為 JSON 檔案，適合系統整合、資料交換或技術分析使用。

## 注意事項

稽核日誌主要用於追蹤系統操作紀錄。可查看的紀錄範圍會依使用者權限而有所不同。

若查詢不到特定紀錄，請確認搜尋條件、篩選條件與時間範圍是否正確。調整條件後，可重新查詢或匯出符合條件的稽核紀錄。
